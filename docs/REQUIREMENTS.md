# 個人日記Webサービス 要件定義

## 概要

個人利用を前提とした、シンプルな日記Webサービス。
単一の実行バイナリを起動するだけで動作する。

---

## 機能要件

### 1. 日記の投稿

- **一日一記事**のみ投稿可能。同一日付への重複投稿は不可（上書き編集は可）
- 投稿時に**日付を選択**する
  - 選択できるのは**本日以前**のみ（未来日は選択不可）
  - 日付をまたいでしまった場合に前日分を投稿できるよう、過去日投稿を許容する
- 本文は**プレインテキスト**形式
  - 本文中に含まれるURLは、表示時に自動的に**クリッカブルリンク**へ変換する
- **画像のアップロード**が可能（複数枚）
  - 対応形式: JPEG, PNG, GIF, WebP
  - 画像はサーバ側のローカルディレクトリに保存し、日記と紐付ける
- 投稿後は内容を**編集・削除**できる

### 2. 日記の閲覧

- トップページに日記の**一覧**を表示（新しい順）
  - 日付・本文の冒頭プレビュー・サムネイル画像（あれば）を表示
  - ページネーション（例: 10件/ページ）
- 日記をクリックすると**詳細画面**へ遷移
  - 本文全文・添付画像を表示
  - URLはリンクとして表示

### 3. 検索

- キーワードによる**全文テキスト検索**（SQLiteのLIKE検索で実装、FTSは不要）
- **日付範囲**での絞り込みも可能（From/To指定）
- 検索結果は日付降順で表示

### 4. データのエクスポート

- **全件エクスポート**または**期間指定エクスポート**で、日記データをZIPファイルとしてダウンロードできる
- ZIPの内容:
  - テキストは `yyyyMMdd.txt` 形式（インポートと対称的な構造）
  - 添付画像は `images/yyyyMMdd/` サブディレクトリに格納
  - 例:
    ```
    diary_export_20240101_20241231.zip
    ├── 20240101.txt
    ├── 20240315.txt
    └── images/
        ├── 20240101/
        │   ├── photo1.jpg
        │   └── photo2.png
        └── 20240315/
            └── photo3.jpg
    ```
- エクスポート画面でFrom/Toを指定して期間を絞り込み可能（未指定時は全件）
- ZIPのファイル名は `diary_export_yyyyMMdd_yyyyMMdd.zip` 形式（例: `diary_export_20240101_20241231.zip`）
- **単一記事エクスポート**: 詳細画面から特定日の日記を `yyyyMMdd.txt` として個別ダウンロードも可能（テキストのみ）

### 5. テキストファイルインポート

#### 5-1. 単一ファイルインポート

- `yyyyMMdd.txt`（例: `20240315.txt`）形式のファイルをアップロードすると、ファイル名に対応した日付の日記として**自動登録**される
- インポート仕様:
  - ファイル名から日付を解析する（不正なファイル名はエラー）
  - ファイルの中身がそのまま本文となる（UTF-8テキスト）
  - **対象日に既存エントリがある場合**は上書き確認ダイアログを表示し、ユーザが選択する
  - インポートは画像なし（テキストのみ）
- ファイルサイズ上限: **10MB**
  - 日記テキストとして実用上十分な余裕を持たせた値。個人のローカル利用を前提とするため、一般的なWebサービスより緩い設定としている

#### 5-2. ZIPファイル一括インポート

- ZIPファイルをアップロードすると、ZIP内の `yyyyMMdd.txt` ファイルを一括で日記として登録できる
- ファイル処理仕様:
  - ZIPの内部ディレクトリ階層は無視し、**ファイル名のみ**を見て処理する
  - ファイル名が `yyyyMMdd.txt` 形式かつ有効な日付であれば登録対象とする
  - 不正なファイル名のファイルは**静かに無視**する（`.DS_Store` 等の混入を想定）
  - **対象日に既存エントリがある場合はスキップ**（上書きしない）
  - インポートは画像なし（テキストのみ）
  - ファイルの中身はUTF-8テキストとして扱う（Shift-JIS等には非対応）
- 処理結果はレスポンスで詳細に返す:
  - 登録成功件数
  - **有効なファイル名だが登録できなかった**ファイルの一覧とその理由（例: `already_exists`）
  - フロントエンドで登録できなかったファイルの情報を画面に表示する
- セキュリティ:
  - ZIPファイルをディスクに展開せず、ストリームとして処理する
  - ファイル名のパス情報は完全に無視することでZip Slip攻撃を防ぐ
- ファイルサイズ上限: **100MB**
  - 長期間の日記を一括インポートする用途を想定し、余裕を持たせた値。個人のローカル利用を前提とするため、一般的なWebサービスより緩い設定としている

## 非機能要件

| 項目 | 内容 |
|---|---|
| ユーザ認証 | 不要（個人専用サービス） |
| セキュリティ | 最低限（ローカルネットワーク等での利用を想定） |
| 性能要件 | 特になし（ユーザ1名） |
| 可用性 | 特になし |
| デプロイ | 単一実行バイナリを起動するだけで動作すること |

## 技術スタック

### バックエンド

| 要素 | 採用技術 | 備考 |
|---|---|---|
| 言語 | **Go** | 必須 |
| Webフレームワーク | `net/http` (標準ライブラリ) または `chi` | 軽量ルーター |
| DB | **SQLite** | `modernc.org/sqlite`（Pure Go・CGo不要）を採用し単一バイナリを実現 |
| DBマイグレーション | `golang-migrate` または 手動SQL | 起動時に自動マイグレーション |
| 静的ファイル配信 | Go `embed` パッケージ | フロントエンドビルド成果物をバイナリに埋め込む |

### フロントエンド

| 要素 | 採用技術 | 備考 |
|---|---|---|
| 言語 | **TypeScript** | 必須 |
| ビルドツール | **Vite** | バンドル・開発サーバ |
| UIフレームワーク | **React** | コンポーネントベースのUI |
| スタイリング | **Tailwind CSS** | ユーティリティファースト |
| HTTPクライアント | `fetch` API | 標準 |

### データ保存

| データ | 保存先 |
|---|---|
| 日記テキスト・メタ情報 | SQLite DB ファイル（例: `diary.db`） |
| 添付画像 | ローカルディレクトリ（例: `./data/images/`） |

---

## データモデル

### `entries` テーブル

| カラム | 型 | 説明 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | 自動採番 |
| `entry_date` | TEXT (YYYY-MM-DD) | 日記の日付（UNIQUE） |
| `body` | TEXT | 本文 |
| `created_at` | TEXT (ISO8601) | 作成日時 |
| `updated_at` | TEXT (ISO8601) | 更新日時 |

### `images` テーブル

| カラム | 型 | 説明 |
|---|---|---|
| `id` | INTEGER PRIMARY KEY | 自動採番 |
| `entry_id` | INTEGER | `entries.id` への外部キー |
| `filename` | TEXT | 保存ファイル名（UUID等） |
| `original_name` | TEXT | 元のファイル名 |
| `order` | INTEGER | 表示順 |
| `created_at` | TEXT (ISO8601) | 作成日時 |

---

## API設計（REST）

| メソッド | パス | 説明 |
|---|---|---|
| GET | `/api/entries` | 日記一覧（ページネーション・検索対応） |
| POST | `/api/entries` | 新規投稿 |
| GET | `/api/entries/:date` | 特定日の日記取得（date: YYYY-MM-DD） |
| PUT | `/api/entries/:date` | 特定日の日記更新 |
| DELETE | `/api/entries/:date` | 特定日の日記削除 |
| POST | `/api/entries/:date/images` | 画像アップロード |
| DELETE | `/api/images/:id` | 画像削除 |
| POST | `/api/import` | txtファイルインポート（単一ファイル） |
| POST | `/api/import/zip` | ZIPファイル一括インポート |
| GET | `/api/export` | ZIPエクスポート（クエリパラメータ: from, to） |

## 画面構成

```
/                   トップ（日記一覧）
/new                新規投稿フォーム
/:date              特定日の日記詳細（例: /2024-03-15）
/:date/edit         日記編集フォーム
/search             検索結果ページ（クエリパラメータ: q, from, to）
/import             テキストファイルインポート
/export             データエクスポート（期間指定ZIP）
```

### UI方針

- ブログサービス風の落ち着いたデザイン
- ヘッダー: サービス名・ナビゲーション（一覧/新規投稿/インポート/エクスポート）・検索バー
- 一覧: カード形式、日付・テキストプレビュー・サムネイル
- 詳細: タイポグラフィを重視した読みやすいレイアウト
- フォーム: 日付ピッカー（カレンダーUI）・テキストエリア・ドラッグ&ドロップ対応の画像アップロードエリア

## 起動方法（最終形）

```bash
# ビルド（フロントエンドをバイナリに埋め込む）
cd frontend && npm run build
cd .. && go build -o diary .

# 起動
./diary
# → http://localhost:8080 でアクセス可能
```

設定はコマンドライン引数または環境変数で指定可能（ポート番号、データディレクトリ等）。
